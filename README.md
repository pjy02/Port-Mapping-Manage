> 🚀 **v5.0 安全架构更新** - 使用统一规则模型和项目专属持久化服务管理 TCP/UDP、IPv4/IPv6 端口映射，不再接管系统完整防火墙规则。

---

## 📑 项目简介

本脚本旨在简化 Linux 服务器上的大范围端口转发配置与管理，特别适用于 **Hysteria2 / Xray / V2Ray / iperf** 等需要映射大量端口的应用场景。v5.0 将规则、持久化和卸载统一到明确的项目资源边界内。

### 🌟 核心特性

* **🎯 智能化管理**：提供直观的交互界面，无需记忆复杂命令
* **🔄 双栈支持**：完整支持 IPv4 / IPv6 双协议栈
* **⚡ 协议灵活**：支持 TCP / UDP，智能协议检测和验证
* **📦 批量操作**：导入时先完整预验证，再单次备份和批量执行，任一失败自动回滚
* **📊 实时监控**：按 TCP/UDP、IPv4/IPv6 精确统计流量和检查监听状态
* **🛡️ 安全可靠**：增强的输入验证、自动备份、智能恢复机制
* **🔧 只读诊断**：全面检查系统、规则和监听状态，诊断过程不会隐式修改系统
* **⚙️ 专属持久化**：只恢复带项目标记的规则，不覆盖 Docker、UFW、firewalld 等第三方规则
* **🚀 性能优化**：缓存机制、批量处理，大幅提升执行效率
* **🌐 广泛兼容**：支持 Debian / Ubuntu / CentOS / Rocky / AlmaLinux 等主流发行版

## 🚀 快速开始

> 安装器会校验发布清单以及所有安装负载。不要直接使用未经校验的 `curl | bash`。

```bash
VERSION=v5.0.0
BASE="https://raw.githubusercontent.com/pjy02/Port-Mapping-Manage/${VERSION}"
curl --proto '=https' --tlsv1.2 -fLO "$BASE/install_pmm.sh"
curl --proto '=https' --tlsv1.2 -fLO "$BASE/install_pmm.sh.sha256"
sha256sum -c install_pmm.sh.sha256
sudo bash install_pmm.sh
```

安装器还会使用内置的受信任清单摘要验证 `release-manifest.sha256`，再分别验证主脚本和启动器。任一摘要或语法检查失败都会中止安装。

## 🔧 新增映射示例

1. 选择 **1. 新增端口映射**
2. 按提示依次输入：
   * 起始端口：`6000`
   * 终止端口：`7000`
   * 本地服务端口：`3000`
   * 协议：`1`（TCP）或 `2`（UDP）
3. 确认后即刻生效，可在 **3. 查看当前映射规则** 中验证。

## 📂 批量导入示例文件

统一规则模型包含 IP 版本、协议、起止端口和目标端口：

```text:sample_rules.conf
# 格式: IP版本|协议|起始端口|结束端口|目标端口
4|udp|6000|7000|3000
4|tcp|8000|9000|4000
6|udp|10000|12000|5000
```

旧版 `start_port:end_port:service_port` 格式仍可导入，导入时按当前 IP 版本和 UDP 协议处理。

导入采用事务流程：全部记录和冲突一次性预验证，通过后只创建一次批次前备份，再顺序执行；任一规则失败都会恢复批次前状态，不会保留半完成配置。

## 🛡️ 备份与恢复

* **自动备份**：每次更改前只备份项目规则模型到 `backups/`，默认保留 10 份
* **选择性清理**：在备份管理菜单输入序号（支持空格/逗号等任意分隔）或输入 `all` 一键清空
* **恢复备份**：只替换带项目标记的规则，不覆盖系统或第三方防火墙规则

## ⚙️ 持久化边界

v5.0 使用 `/etc/port_mapping_manager/rules.db` 作为唯一持久化数据源，并通过 `pmm-rules.service` 幂等恢复规则。启动脚本只做只读验证，不会自动安装软件、创建服务或修改启动项；所有修复都必须在菜单 11 中显式确认。

新版本不会写入或删除 `/etc/iptables/rules.v4`、`rules.v6`，也不会创建 crontab、`rc.local` 或网络接口 fallback。显式修复会定向迁移旧版本明确属于本项目的启动项。

## 📊 实时监控

在 **8. 实时监控** 中，可按每条规则的 IP 版本和协议查看累计数据包、字节数、实时速率及目标端口监听状态。诊断报告会持久保存在 `/etc/port_mapping_manager/reports/`，不会随临时文件清理而丢失。

---

## 🆘 故障排查

1. 确认脚本以 **root** 身份执行
2. 检查 `iptables` 是否正常工作：`iptables -L -n`
3. 若规则未生效，尝试重启相关服务并确保本地端口监听
4. 查看日志：`logs/udp_mapping_*.log`

## 🔄 升级指南

菜单中的更新功能只接受安装时保存的信任锚，不再跟随可变的 `main` 分支。升级到新的发布版本时，请重新下载该版本的安装器和安装器校验文件：

```bash
VERSION=v5.0.0
BASE="https://raw.githubusercontent.com/pjy02/Port-Mapping-Manage/${VERSION}"
curl --proto '=https' --tlsv1.2 -fLO "$BASE/install_pmm.sh"
curl --proto '=https' --tlsv1.2 -fLO "$BASE/install_pmm.sh.sha256"
sha256sum -c install_pmm.sh.sha256
sudo bash install_pmm.sh
```

也可以由管理员通过 `PMM_TRUSTED_REF` 和 `PMM_TRUSTED_MANIFEST_SHA256` 显式提供经过独立渠道确认的新信任锚。没有信任锚时更新会失败关闭。

## 🗑️ 卸载脚本

```bash
pmm --uninstall
```

或在主菜单输入 `99` 按提示卸载。卸载只删除带项目 comment 的规则、`pmm-rules.service`、项目配置和启动器；不会禁用系统 `netfilter-persistent`、删除全局防火墙文件或卸载依赖包。
